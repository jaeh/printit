µcms

is a small and simple cms for nodejs, always following the dry kiss principle.

it is built on top of expressjs, swig html templates, stylus css preprocessor and nodemailer.

its open source too, in case you where wondering. you can fork the project on

features:
template based:
configuration and content resides in files. there is no database and no metadata, all content is kept within the views directory of the app.
valid html5 and css3
we wont support browser quirks. if rendering goes wrong its the fault of the browser and its time for developers to force them instead of being forced. we will always conform to the w3c rules for html5 and css3.
jslinted javascript
while jslint doesnt help a lot when it comes to bugs (see mocha below :p) it still helps open source projects keeping organized. it also leads to readable and reusable code and respects javascript standards.
mocha testing
well, so far there are three test functions, one that is testing if the absolutely necessary templates exist physically, one that tests the nodemailer implementation and one that tests if configure.do is excessible. more will follow.
hooks in layout.html
there are more than 20 "blocks" in the layout.html file, allowing the users to overwrite almost everything in their themes as well as adding content before and after all elements. whilst very simple this should allow almost every kind of extension. documentation following soon.
nodejs open source applications used:
expressjs
an almost perfect and very enjoyable web framework for nodejs, in the authors own words: Express is a minimal and flexible node.js web application framework, providing a robust set of features for building single and multi-page, and hybrid web applications.
express-form
awesome form validation and data filtering for express
express-load
plugin to easily load controllers and routes. really reduces clutter.
swig html templating
its just the template language that seemed to be most optimal for our needs, should be extremely easy to replace (jade and hogan examples will follow soon)
stylus css preprocessor
makes css readable. removes all the unnecessary clutter and adds various ways to programmatically create css. should be exchangable with a few lines. (examples of other preprocessors coming soon)
